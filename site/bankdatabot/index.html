    The BankDataBot 🤖

    About

    The BankDataBot is a pre-release/beta test of an automated service that polls the Federal Financial Institution Examination Council's web site and webservice for new filings, refilings, and other frequently changing data. Data updates from the FFIEC site are processed, stored, and utilized to generate tweets at @BankDataBot

    Types of Automated Posts

    Holding Company Data (Y-9 Series)

    BankDataBot collects the daily snapshots of all non-confidential holding company data as [published by the FFIEC and Federal Reserve])(https://www.ffiec.gov/npw/FinancialReport/FinancialDataDownload).

    Published data reflects data fields present on the Y-9, Y-9SP, and Y-9LP reports.

    As of 2022, data is available for all quarters, starting in the 2nd Quarter, 2016. Data is provided as a snapshot due to the fact that holding companies can refile their previously submitted figures.

    Refilings

    BandDataBot automatically downloads each day's data snapshot, then extracts and normalizes field names and data types. The result of this normalization is then stored for subsequent review.

    Following the processing of each day's reporting, the current day's data is compared to the prior day. Data points that changed overnight (with respect to the Bank's RSSD ID, reporting quarter, and data field name) are flagged and collected.

    The changed data points (the "diffs") are then processed for automated posting onto the @BankDataBot twitter account.

    If twitter screen names are associated with the holding company, or the bank, if the bank is solely owned by one bank holding company, are included in the automated tweest.

    Tweet Content

    The text of the holding company refiling tweets contain these data points:

    • The name, headquartered city, and headquartered state of the holding company
    • The RSSD ID of the holding company
    • The date of the refiling
    • The count of data series that have been changed from the prior quarter
    • The count of quarters over which those data series have been changed
    Tweet Data

    Tweet data is provided as a gzip-ed JSON file with the following structure:

    • id_rssd (string): The ID (RSSD) of the holding company
    • name (string): The name of the holding company
    • city(string): The city in which the holding company is headquartered
    • state (string): If the holding company is headquartered in the US, the US State; otherwise, null.
    • country (string): The country in which the holding company is headquartered
    • attribs_as_of (string): The date and time (in UTC) when the above attributes associated with the RSSD were collected from the FFIEC.
    • changed_data_as_of (string): The date when the data associated with the holding company was collected from the FFIEC. The FFIEC does not post the specific time at which new data is posted, so only the date is provided. Data provided in YYYY-MM-DD format.
    • changed_data list(map):
    • id_rssd (string)
    • mdrm (string): The ID of the data point
    • data_type (string): Indicator if the data is a string, numeric, or boolean
    • as_of_qtr (integer): The numeric quarter (1,2,3,4) of the data's reporting period
    • as_of_year (integer): The numeric year of the data's reporting period
    • latest_date (string): Data provided in YYYY-MM-DD format, the last reported data point as of the date.
    • previous_date (string): Data provided in YYYY-MM-DD format, the previous reported data point as of the date.
    • latest_numeric_val (numeric): The numeric data of the last reported figure
    • previous_numeric_val (numeric): The numeric data point of the previously reported figure
    • latest_string_val (string): The string data point of the last reported data point
    • previous_string_val (string): The string data point of the previous reported data point
    • latest_bool_val (string): The boolean data point of the last reported data point
    • previous_bool_val (string): The boolean data point of the previous reported data point
    • field_name (string): The human-readable name of the data field

    FFIEC 031, 041, and 051 Regulatory Data

    BankDataBot monitors filings published through the FFIEC's Webservice interface, an API-based service that provides the individual filings of FFIEC 031, 041, and 051, on a "live" basis.

    In other words, as soon as a financial institution submits its data to the FFIEC, and as soon as that data is validated, the data may be downloaded from the Webservice API.

    The service, however, is pull-based, meaning that in lieu of notices "pushed" by the FFIEC to subscribers, the API must be queried for information.

    BankDataBot conducts these queries on an automated basis, collects facsimiles of the submitted regulatory reports, and joins relevant metadata to the resultant dataset.

    Refilings

    Refiling postings are tweeted when BankDataBot identifies a filing of a regulatory report that is updated from a previous filing.

    For example, a bank may file its initial quarterly report on April 15th for the March 31st (1st Quarter) reporting date, but then updates its filing, sending new data on July 15th.

    The FFIEC Webservice does not maintain prior copies of filings, therefore when a bank files an updated quarterly filing, the previous filing disappears.

    The BankDataBot maintains a rolling record of all new and previous filings, and when a refiling is detected, conducts a comparison between the prior and the updated filing.

    FAQs

    1. Is a reason provided for an institution updating their regulatory filing? The reason for a refiling is not included in the submitted data; the institution would need to be contacted directly to ascertain the reason. Reasons may include, errors, financial report restatements, and responses to regulators' mandates.

    2. How long does it take for a refiling to show up on the BankDataBot? Refilings should show up within 24 hours, however, this lead time should decrease as processes are finalized.

    Tweet Content

    The tweets contain the following info:

    • Date of updated report filing
    • Date of previous report filing
    • Name of institution
    • #RSSDXXXXXX a hashtag with the RSSD (ID) of the institution. Clicking this hashtag will show other tweets on BankDataBot (or elsewhere) related to this institution.
    • If there is a twitter screen name associated with the institution, that name is included in the tweet.
    • The name of the refiled report (031, 041, or 051)
    • The number of data series changed between the refiled report and the last filed report.
    • A link to a gzipped-JSON file containing the original filing, updated filing, and changed data fields.
    Tweet Data
    • Each refiling contains a link to the source data from which the tweet text was generated.
    • The gzipped JSON file may be decompressed natively in a Linux-based OS or MacOS. Windows users can use the 7-Zip utility to decompress the file.
    • The JSON file is a machine-readable file; however, Firefox is recommended for a human-friendly hierarchial interface to explore file content.
    • Schema:
    • id_rssd (integer): The FFIEC-assigned identification number for the reporting financial institution.
    • bank_name (string): The legal name of the bank, as recorded by the FFIEC. This name may be slightly different from the "human-friendly" name included in the tweet text.
    • city(string): The city of the institution's headquarters
    • state (string): The two-character Postal Service U.S. state code indicating the location of the bank's headquarters.
    • as_of_quarter (int): The reporting quarter referenced by the filing. 1 = 1st Quarter ending on 3/31, 4 = 4th Quarter ending on 12/31.
    • as_of_year(int): The 4-digit reporting year
    • rpt_name (string): The name of the report filed by the respective institution. Options are FFIEC 031, FFIEC 041, FFIEC 051.
    • last_filing_date (string): The timestamp for the date and time of the refiling in `YYYY-MM-DD HH:MM Z" format. Note that the timestamp timezone is shifted to UTC from the native time zone in Washington, DC.
    • previous_filing_date (string): The timestamp for the immediately previous filed financial data, in the same format as the last_filing_date.
    • latest_data & previous_data (list[map]): The regulatory filing data collected with the latest filing, and the regulatory data filing from the prior filing, respectively.
      • mdrm (string): The ID of data field
      • rssd (string): The ID (RSSD) of the reporting institution
      • quarter (string): The institution's reporting quarter in MM/DD/YYYY format. The MM/DD will have the values of 3/31, 6/30, 9/30, or 12/31.
      • int_data (integer): If the data series reports data in integer format, this field will contain that value.
      • float_data (float / double): If the data series reports data as a floating point (double) format, this field will contain that value.
      • bool_data (boolean): If the data series reports data in boolean format, this field will contain that value.
      • str_data (string): If the data series reports data in string format, this field will contain that value.
      • data_type (string): Indicates if the data series reports an int, float, bool, or str value.
      • field_name (string): The name of the data field, as reported by the MDRM Data Dictionary.

    • changed_fields (list[map]):

      • ...same fields as prior, with latest or prior appended to the respective data fields.

    • change_type (string): Indicates the name of the data schema; will always be the value refiling

    General FAQs

    • Why is the output data between the bank and the holding company filings different?

      • The source data provided by the FFIEC for banks and for holding companies have different formats and metadata. While some effort has been taken to normalize the output datasets between the two sources, more work is needed. Eventually, the output data should be the same.

    • Where can I find information on updates to the format or content of tweet text or data?

    • The twitter account @BankDataBotKeep will post the latest announcements related to the BankDataBot.

    • Can I download a bulk version of the tweets or output data?

    • Yes, bulk data download is available as a service. Please contact michael@civicobj.com to inquire.

    • Is an alternative distribution method available available for the content contained in the tweets?

    • Yes, alternative distribution, including webhooks, Kafka endpoints, or AWS SQS queues, methods are available as a service. Please contact michael@civicobj.com to inquire.
